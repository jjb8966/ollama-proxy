# 구현 계획: ollama-proxy 기능 개선

**작성일:** 2026-03-18
**관련 연구:** research_260318.md

---

## 1. 개요

 네 가지 기능을 ollama-proxy에 추가합니다:

1. **응답 끊김 원인 추적 로그** (최우선)
2. Context length 및 max token 체크 로직
3. 쿼터(Quota) 조회 기능
4. 계정 선택 로직 개선

---

## 2. 구현 단계

### [0단계] 응답 끊김 원인 추적 로그 추가 (최우선)

**목표:** 응답 끊김 문제 원인을 추적하기 위한 상세 로깅

#### 2.0.1 스트림 생명주기 로깅

**작업 내용:**
- 요청 시작/종료 시간 로그
- 스트림 첫 청크 수신 시간

**로그 형식:**
```python

logger.info(f"[Stream] ⏱️ 첫 청크 | latency={first_chunk_latency:.3f}s")
logger.info(f"[Stream] ✅ 종료 | model={model} | finish_reason={finish_reason} | duration={total_duration:.3f}s")
logger.warning(f"[Stream] ⚠️ 5초간 수신 없음 | model={model}")
```

#### 2.0.3 finish_reason 상세 분석

**작업 내용:**
- finish_reason별 상세 로그
- "length" 종료 시 원인 분석 로그
- 컨텍스트 길이, max_tokens, 실제 토큰 수 비교

**로그 형식:**
```python
if finish_reason == "length":
    logger.warning(
        f"[Finish] ⚠️ LENGTH 종료 | model={model} | "
        f"max_tokens={max_tokens} | estimated_tokens={estimated} | "
        f"이유: max_tokens 도달 또는 context 초과 가능성"
    )
elif finish_reason == "stop":
    logger.info(f"[Finish] ✅ STOP 종료 | model={model}")
elif finish_reason == "tool_calls":
    logger.info(f"[Finish] 🔧 TOOL_CALLS 종료 | model={model} | tools={len(tool_calls)}")
```

**파일:**
- `src/handlers/response.py`

**상태:** ⏳ Pending

#### 2.0.4 예외 상세 로깅

**작업 내용:**

**로그 형식:**
```python

    logger.error(f"[Exception] ⏱️ 타임아웃 - upstream 응답 지연 가능성")
elif isinstance(e, requests.exceptions.ConnectionError):
    logger.error(f"[Exception] 🔌 연결 오류 - 네트워크 또는 upstream 문제")
```

### [2단계] 쿼터 조회 기능

**목표:** CLI 및 API로 현재 쿼터 상태 조회 가능

#### ⚠️ 선행 작업: antigravity-proxy 쿼터 조회 API 추가

**현황:** antigravity-proxy는 내부적으로 `_fetch_antigravity_quota()` 메서드로 쿼터를 조회하지만, **외부에 API로 노출하지 않음**.

**필요 작업 (antigravity-proxy):**
1. `GET /v1/quota` 또는 `GET /v1/accounts/status` API 엔드포인트 추가
2. 계정별 쿼터 상태 (remainingFraction, resetTime) 반환
3. 모델군별 집계 (claude, gemini-pro, gemini-flash)

**더미 API 구현 (개발용):**
```python
# antigravity-proxy/app.py
@app.route('/v1/quota', methods=['GET'])
def get_quota():
    """계정별 쿼터 상태 조회 API"""
    # 현재는 더미 응답 반환
    # 실제 구현 시 _fetch_antigravity_quota() 호출
    return jsonify({
        "accounts": [
            {
                "email": "account@example.com",
                "antigravity": {
                    "claude": {"remainingFraction": 0.85, "resetTime": "..."},
                    "gemini-pro": {"remainingFraction": 0.42, "resetTime": "..."},
                    "gemini-flash": {"remainingFraction": 0.70, "resetTime": "..."}
                }
            }
        ]
    })
```

#### 2.3.2 쿼터-aware 선택 알고리즘

**작업 내용:**
- tier threshold 기반 선택 (70% -> 40% -> 10%)
- 점수 계산: 쿼터 잔량 + 최근 선택 + 건강도

**알고리즘:**
```
1. 사용 가능한 키 필터링 (not rate-limited, not exhausted)
2. Tier 1 (quota >= 70%) 중 점수最高 선택
3. Tier 2 (quota >= 40%) 중 점수最高 선택
4. Tier 3 (quota >= 10%) 중 점수最高 선택
5. Fallback: 가장 빨리 복귀 가능한 키 대기 후 재시도
```
