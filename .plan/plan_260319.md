# Plan: fix Claude Code OMC marketplace mismatch

Date: 2026-03-19

## Goal

Remove the Claude Code plugin error caused by resolving `oh-my-claudecode` against the wrong marketplace identifier.

## Confirmed root cause

- User-scope config uses marketplace key `oh-my-claudecode`.
- Actual marketplace manifest name is `omc`.
- Claude Code expects plugin references to use the marketplace identifier from the manifest.

## Implementation steps

1. Update `/home/jjb/.claude/settings.json`.
   - Change `extraKnownMarketplaces.oh-my-claudecode` to `extraKnownMarketplaces.omc`.
   - If plugin should be auto-enabled, add `enabledPlugins["oh-my-claudecode@omc"] = true`.

2. Repair marketplace registry metadata if still stale.
   - Ensure `/home/jjb/.claude/plugins/known_marketplaces.json` contains `omc` instead of stale `oh-my-claudecode` mapping.
   - Remove stale orphaned `oh-my-claudecode` marketplace folder only if it is confirmed unused after repair.

3. Refresh Claude Code plugin state.
   - Run the appropriate plugin reload/update flow.
   - Re-open the plugin UI or rerun the listing command to confirm the error is gone.

4. Verify.
   - Confirm marketplace list recognizes `omc`.
   - Confirm installed/discover view no longer reports `Plugin 'oh-my-claudecode' not found in marketplace 'oh-my-claudecode'`.

## Risks

- These are user-scope config files outside the repo, so incorrect edits could affect Claude Code globally.
- Deleting stale plugin directories is irreversible without reinstall, so deletion should be avoided unless clearly necessary.

## Approval needed

- This plan requires editing `/home/jjb/.claude/settings.json` and possibly `/home/jjb/.claude/plugins/known_marketplaces.json`, which are outside the workspace.
- Explicit user approval is required before making those changes.
