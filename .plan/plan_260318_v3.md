# Plan 260318 v3

## 목표
Anthropic `/v1/messages` 에서 툴 사용 후 후속 요청이 실패할 때, 스트리밍 클라이언트가 "응답이 끊겼다"가 아니라 명시적인 Anthropic 호환 오류를 받도록 수정합니다.

## 작업 계획

1. 오류 반환 계약 정리 (Done)
- `ChatHandler.handle_chat_request()` 가 사전 검증 실패를 일반 성공 payload(dict) 와 구분 가능한 형태로 반환하도록 구조를 정리합니다.
- 최소 기준은 route 가 "정상 upstream 응답"과 "사전 검증/내부 오류"를 안전하게 구분할 수 있어야 합니다.

2. Anthropic route 에러 매핑 보강 (Done)
- `src/routes/anthropic.py` 에서 context 초과 등 사전 검증 실패를 Anthropic 호환 에러 바디로 변환합니다.
- `stream=True` 요청이어도 스트림 시작 전 실패는 `application/json` 에러 응답과 적절한 status code 로 반환합니다.
- 성공 경로와 오류 경로 로그를 명확히 분리합니다.

3. 비정상 dict 응답 방어 로직 추가 (Done)
- `handle_non_streaming_response()` 에 들어오면 안 되는 dict 형태가 유입될 경우 조용히 성공 처리하지 않도록 방어합니다.
- 잘못된 포맷은 변환 실패로 처리해 계약 위반을 빠르게 드러내도록 합니다.

4. 회귀 테스트 추가 (Done)
- stream 요청 + context 초과 시 Anthropic error 형식과 status code 검증
- non-stream 요청 + context 초과 시 동일하게 Anthropic 호환 오류 검증
- 정상 tool_use/tool_result 흐름 기존 테스트가 깨지지 않는지 확인

5. 검증 (Done)
- 관련 단위 테스트 실행
- 가능하면 로컬에서 context 초과 입력으로 route 레벨 응답 형식 확인

## 완료 기준
- tool_use 이후 context 초과가 발생해도 클라이언트가 조용한 끊김 대신 명시적인 오류를 받음
- `Anthropic non-streaming response success` 가 오류 dict 에 대해 잘못 찍히지 않음
- 정상 tool_use 스트리밍 경로는 그대로 유지됨
