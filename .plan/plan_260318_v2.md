# Claude Code 도구 호출 중단 수정 계획 (2026-03-18)

## 목적
- Claude Code가 커스텀 `ANTHROPIC_BASE_URL`을 통해 이 프록시의 `/v1/messages`를 사용할 때, `bash` 등 도구 호출 뒤 후속 응답이 이어지지 않는 문제를 해결합니다.

## 현재 판단
- 1순위 원인: `src/handlers/anthropic.py`의 `_normalize_messages()`가 `tool_result`를 OpenAI 호환 메시지로 바꾸는 과정에서 순서를 깨뜨립니다.
- 2순위 원인: assistant `text`와 `tool_use`를 두 개의 assistant 메시지로 중복 생성합니다.
- 별도 운영 이슈: 일부 `OLLAMA_API_KEYS`가 `401`을 일으켜 간헐적 끊김을 악화시킵니다.

## 실행 계획
- [x] `AnthropicHandler._normalize_messages()`를 순서 보존 방식으로 재작성
  - Anthropic `content` 배열을 블록 순서대로 읽습니다.
  - assistant `text`와 `tool_use`는 가능한 한 하나의 assistant 메시지로 조립합니다.
  - user `tool_result`는 대응 assistant `tool_calls` 바로 뒤에 오는 OpenAI `tool` 메시지로 출력합니다.
  - user text는 `tool_result` 뒤에 별도 user 메시지로 추가하되, 원래 순서를 유지합니다.
- [x] 도구 호출 회귀 테스트 추가
  - assistant `tool_use` -> user `tool_result`
  - assistant `text + tool_use` -> user `tool_result`
  - user `[tool_result, text]`
  - 일반 text-only 메시지
- [x] 기본 동작 검증
  - `python3 -m unittest -v tests.test_anthropic_handler`로 메시지 순서 회귀 테스트를 통과했습니다.
  - `python3 -m compileall src tests`로 수정 코드와 테스트 파일의 문법 검증을 통과했습니다.
- [ ] Anthropic 스트림 실동작 검증
  - 실제 Claude Code 또는 동등한 `/v1/messages` 요청으로 `tool_use` 발생 시 `stop_reason=tool_use`까지 정상 반환되는지 확인합니다.
  - `tool_result` 반영 후 assistant 후속 응답이 이어지는지 확인합니다.
- [ ] 401 키 격리 정책 보강 여부 결정
  - 구조 버그 수정 후에도 간헐 끊김이 남으면 401 키 임시 제외 로직을 추가합니다.

## 구현 원칙
- 아직 구현하지 않습니다.
- 사용자가 “구현해”, “전부 구현해라”라고 명시하기 전까지는 코드 수정 없이 문서와 원인 분석만 유지합니다.

## 기대 결과
- Claude Code에서 `bash` 또는 기타 도구 실행 후, tool result가 올바른 대화 히스토리로 upstream에 전달됩니다.
- 이후 assistant가 다음 사고와 응답을 정상적으로 이어갑니다.
- 간헐적 401은 별도 운영 이슈로 분리해 다룰 수 있게 됩니다.
