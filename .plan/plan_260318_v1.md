# ollama-proxy 응답 끊김 조사 계획 (2026-03-18)

## 목적
- Claude Code에서 체감하는 "답변이 중간에 끊김" 현상이 `tool` 문제인지, 특정 `OLLAMA_API_KEYS` 토큰 문제인지 구분합니다.

## 현재 결론
- `tool` 자체가 직접 깨진 증거는 아직 없습니다.
- 로그상 명시적 실패는 `OllamaCloud` upstream `401 Unauthorized`이며, 실패 키는 현재 2개로 좁혀집니다.
- Anthropic 스트리밍 경로는 중간 단절 예외를 충분히 로깅하지 않으므로, 체감 장애 대비 관측성이 부족합니다.

## 실행 계획
- [ ] 문제 토큰 분리 검증
  - 로그에 반복적으로 나타난 2개 마스킹 키를 실제 키 목록과 대조합니다.
  - 해당 키를 임시 제외한 뒤 동일 요청 패턴에서 끊김 재현 여부를 확인합니다.
- [ ] Anthropic 스트림 관측성 보강
- [x] Anthropic 스트림 관측성 보강
  - `src/handlers/anthropic.py`에 스트림 시작, 첫 청크, `[DONE]`, 예외, 종료 로그를 추가합니다.
  - 요청별 상관관계 확인용 request id 또는 message id 로그를 같이 남깁니다.
- [ ] 401 키 격리 개선
- [x] 401 키 격리 개선
  - `401` 발생 시 해당 키를 실패 처리하여 일정 시간 회전 대상에서 제외하는 방안을 검토합니다.
  - 현재 `StandardApiClient` 경로에서 `mark_key_failure()`가 호출되지 않는 부분을 보강합니다.
- [x] 재현 확인
  - `tools=30` 요청과 `tools=1` 요청을 각각 반복해, 끊김이 tool 개수와 관련 있는지 다시 검증합니다.

## 구현 대기
- 실패 키 추적을 위한 로깅 및 라벨 매핑은 구현했습니다.
- Anthropic 스트림 관측성 보강도 구현했습니다.
